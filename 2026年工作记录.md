### 2026年5月21日

**项目：** 玉禾田深圳

**紧急重要度：** 2026-05

BUG/反馈修改[2h]：WDP 云渲染请求 Pending 问题 
 一、问题
Vue3 + Vite 项目中，进入 3D 场景执行 `addPOI`（添加 POI 点位）和 `flyCameraTo`（飞镜头）时，`dtp-api.51aes.com`（WDP 云渲染 API）的请求在 Network 面板中 Pending 约 2秒才真正发出，导致 POI 加载和镜头动画明显卡顿。换到无痕浏览器后完全正常。
二、排查过程
1.代码层排查
对 POI 加载逻辑进行了多轮优化与验证，包括将并行请求改为串行 await、移除 customData 等非必要字段、重构 loadGroup 加载逻辑等，确认问题与业务代码结构、请求并发及数据量无关。
2.环境对比验证
通过控制变量法对不同运行环境进行对比测试：
将 WDP 调用抽离至独立 HTML 页面后请求秒出；
切换至 Vue2 项目后表现正常；
使用 Chrome 无痕模式后完全正常。
通过多环境差异对比，初步锁定问题与浏览器环境。
3.网络层排查
进一步排查浏览器网络及连接问题，包括：
禁用 STOMP 与大疆直播相关 WebSocket；
页面刷新时增加 Renderer.Stop() 主动释放云渲染资源；
使用 chrome://net-internals/#sockets 清理浏览器连接池。
验证后问题仍存在，排除了 WebSocket 占用及 HTTP 连接池耗尽问题。

三、根因
最终通过逐个禁用 Chrome 扩展定位问题来源，确认以下扩展均会对请求产生额外延迟：
Tampermonkey：请求发送前执行 @match 规则匹配；
Vue.js devtools beta：拦截请求并注入 Vue 调试探针；
Authenticator：监听请求检测 2FA 页面。
多个扩展在 chrome.webRequest.onBeforeRequest 阶段的处理耗时叠加后，导致请求实际发送前出现约2秒 Pending。
四、解决方案
开发环境禁用 Tampermonkey、Vue devtools beta、Authenticator 等扩展；保留 beforeunload 中调用 Renderer.Stop() 的资源释放逻辑。问题处理后，WDP 请求恢复即时发送，POI 加载与镜头飞行动画卡顿问题消失。
五、经验总结
1. 无痕正常 + 正常浏览器异常 = 先查扩展。这条规律可以跳过大量代码和网络层的无效排查，直接定位到浏览器环境问题。
2. 控制变量法是最有效的排查手段。单独 HTML 秒出、Vue2 正常、无痕正常 — 三组对比锁定了"浏览器扩展 + 特定 URL 模式"这个交叉条件。
3. 多个扩展的延迟会叠加。单个扩展可能只慢几百毫秒不易察觉，多个叠加后才暴露问题。
4. 开发环境建议使用干净的浏览器 Profile。与日常浏览器隔离，避免扩展、缓存、Cookie 等干扰开发调试。
5. 排查网络问题时，先看 DevTools Timing 选项卡的 Stalled 字段。Stalled 时间长指向浏览器层面（扩展/缓存），TTFB 长指向服务端，Queueing 长指向连接池。
